Estrutura de documentos e sites

Seções básicas de um documento

  1. Cabeçalho (header)
    2. Normalmente é uma grande faixa ou algo destacado no topo do site com o logo. Os principais tópicos do site ficam ali
  2. Barra de navegação (nav)
    3. Geralmente uma barra fixa com botões de menu ou links. Esse conteúdo permanece consistente de uma página para outra. Apesar de muitos webdesigners gente considerar como parte do header, isso não é um requisito.
  3. Conteúdo principal (main)
    4. É a área central com a informação que a página vai transmitir. Isso muda em todas as páginas, diferente do resto das seções que podem ser mantidas.
  4. Rodapé (footer)
    5. Uma faixa na parte inferior da página, geralmente contém letras pequenas, como copyright ou informações contratuais. Semelhante ao cabeçalho, porém com informações não relevantes para o leitor, porém necessárias para questões processuais ou SEO (Otimização dos mecanismos de buscas).

Elementos do layout html em mais detalhes

  • main
    • Usar apenas uma vez por página. Não aninhar ele dentro de outro elemento sem ser o body. Ele é o conteúdo principal da sua página.
  • article
    • Inclui um artigo ou citação referente a passagem que está sendo citada. Uma explicação que faz sentido por si só sem depender dessa passagem.
  • section
    • É uma seção para agrupar uma parte da página para diferentes usos, é bom para usar com css.
  • aside
    • É para conteúdo que não está relacionado com o principal, mas que podem providenciar informações complementares. (glossário, links relacionados, etc)
  • header
    • Representa um conteúdo introdutório. Se for filho do body, então é um header global da página do site; se for filho de um article ou section, então é definido como header específico daquela seção.
  • nav
    • Geralmente uma fixa com comandos importantes para a navegação daquela página.
  • footer
    • Uma faixa na parte inferior da página, geralmente contém letras pequenas, como copyright ou informações contratuais.

Elementos de layout não semânticos

Elementos que não tem semântica (indicação de significado) para o html.

  • span
    • O usuário deve dar preferência para usá-lo quando não tiver outras opções. Como uma nota fora de contexto ou uma informação com características css diferenciadas.
  • div
    • Elemento a nível de bloco. Deve-se utilizar quando não conseguir pensar em um bloco semântico melhor e querer adicionar um significado específico. Como um time de pokemon sendo montado em um site de batalhas, vide pokemon showdown.

Observação: É importante dar preferência sempre para usar algo semântico em seu site.

Quebras de linha

  • br
    • Uma quebra de linha em um parágrafo.
  • hr
    • Cria uma literal linha que corta o texto no meio.
    Assim: <hr>
    (possui um hr aqui onde escrevi)

Planejanto um simples site

  1. Muitos sites possuem muitos elementos comuns, como menú de navegação, e o reodapé.
    2. Faça uma anotação de o que você pretende que seja comum em todas as páginas.
  2. Em seguida desenhar um esboço para dar melhor visualização da pretenção.
  3. Escreva uma grande lista com todo o conteúdo que você deseja pôr no site.
    4. Os divida em conjuntos que ficarão juntos e os separe dos outros conjuntos.
  4. Crie balões de informação para saber se está corretamente como em sua ideia.
