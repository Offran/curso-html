Áudio e vídeo

Vídeo

O elemento <vídeo> permite incorporar vídeo.
<video controls > <source src="https://edisciplinas.usp.br/pluginfile.php/5196097/mod_resource/content/1/Teste.mp4" type="video/mp4"><p>

O código acima não funciona em todos os navegadores, pois eles suportam formatos diferentes.

Os recursos acima são:
src: É o caminho para o vídeo que deseja incorporar ao site.
controls: Inclui a interface de controle do navegador ou até mesmo pode ser usado para criar a própria interface usando JS. Uma interface que deve pelo menos ajustar volume e pausar.
O parágrafo dentro de vídeo: Ele é exibido se a página não supoprtar o elemento <vídeo>.

O elemento <source> aponta para sua própria fonte, assim fazendo o vídeo reproduzir primeiro o que for suportado.

Conteúdo de um arquivo de mídia>

Formatos como MP3, MP4 e WebM são chamados de arquivos recipiente.
Arquivo recipiente é o tipo que guarda vários tipos de dados, de forma diferente para cada tipo de arquivo. Os dados são comprimidos e então são retirados parte por parte do áudio e vídeo.

Nota: Embora o MP3 seja muito popular e funcione em praticamente qualquer navegador, não é tudo tão simples, então é interessante sempre pesquisar a compatibilidade do arquivo.

Para maximizar a probabilidade do arquivo de mídia funcionar no seu site para diferentes navegadores, deve-se usar vários formatos.

<video controls> <source src="rabbit320.mp4" type="video/mp4" /> <source src="rabbit320.webm" type="video/webm" /> <p> Your browser doesn't support HTML5 video. Here is a <a href="rabbit320.mp4">link to the video</a> instead. </p> </video>

Outros recursos de vídeos

  • width (largura)
  • height (altura)
  • autoplay (vídeo começa a ser reproduzido imediatamente)
  • loop (vídeo/áudio começa a ser reproduzido novamente assim que termina)
  • muted (mídia reproduzida sem som por padrão)
  • poster (url da imagem será exibida antes da reprodução do vídeo. Geralmente usado como tela inicial ou tela de publi)
  • preload (usado para arquivos grandes em buffer (armanezamento temporário de dados durante sua transferência entre diferentes dispositivos).
    • Esse buffer pode ter três valores:
    "none" = não armazena em buffer o arquivo.
    "auto" = armazena em buffer o arquivo de mídia.
    "metadata" = armazena em buffer apenas os metadados do arquivo. )

Elemento <áudio>

Muito semelhante ao elemento <vídeo>, porém com algumas diferenças.

  • O elemento <audio> não suporta atributos width e height.
  • Não suporta poster.
  • Além desses citados, <audio> suporta praticamente tudo do elemento <video>
