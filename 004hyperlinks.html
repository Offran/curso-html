Hyperlinks

Links são usados para endereçar algo em um texto ou local do site. Eles podem servir para encaminhar o usuário para onde o texto diz, dar uma referência de algo, etc.
Dessa forma: <a href="https://trello.com/b/ncrlYDzg/rpg-naruto-160"> Meu rpg <a>

Guia sobre caminhos

É possível ter mais de um index, desde que não sejam do mesmo diretorio (isso é, não estejam na mesma pasta)

Posso incluir um link dando como referência o meu arquivo.
Por exemplo: Acessar 000tutorial-basico

Posso descer de diretorio, indo para um subdiretório:
Por exemplo: <a href="pasta-exemplo/arquivo-exemplo.html">Para caso tenha uma pasta nesse mesmo diretório e eu queira acessar um arquivo dentro dela.<a>

Voltando para o diretório pai:
Exemplo: <a href="..pasta-exemplo/arquivo-exemplo.html">Eu estaria "recuando" até esse diretório ao invés de "avançando". Esse é um caso onde eu estou no diretório filho/subdiretório e então volto para o pai.<a>

Nota: É possível combinar várias instâncias para entrar ou voltar rapidamente:
Por exemplo: <a href="../../pasta-exemplo2/pasta-exemplo/arquivo-exemplo.html">Eu estaria "recuando" até esse diretório ao invés de "avançando". Esse é um caso onde eu estou no diretório filho/subdiretório e então volto para o pai.<a>

Url relativo e URL absoluto

As diferenças desses dois é que um deles aponta para o index principal do site, como "google.com.br", enquanto outro aponta para um subdiretório, que poderá ser movido pelo programador daquele site.

Práticas recomendadas

  • Use palavras-chave claras para os links.
    • Usuários gostam para uma leitura mais fluida, motores de busca usam para indexar arquivos de destino e usuários podem clicar mais facilmente ao ler tais palavras-chave.
  • Não repita a url para fazer o link, fica feio.
  • Não cite "link" em um link, pela redundância.
  • Mantenha o rótulo do link tão curto quanto o possível, links longos não são estéticos.
  • Seja preciso onde colocar os links e não os coloque toda hora repetidamente.
  • Use links relativos sempre que possível
    • URLs absolutos são procurados na localização real do DNS, então ele será menos eficiente do que uma URL relativa na hora de procurar.
  • Ao vincular algum download ou recurso que não seja html, deixe isso claro.

Use o atributo "download" ao vincular um download.
Por exemplo: <a href="https://download.mozilla.org/?product=firefox-39.0-SSL&os=win&lang=en-US" download="firefox-40-installer.exe">Download Mozilla<a>
